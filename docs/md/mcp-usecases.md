# Artifact Hub — MCP Use Case Matrix

All 7 MCP tools, single-tool uses, multi-step chains, human-in-the-loop flows, and coverage gaps.

**Tools:** `publish_artifact` · `list_artifacts` · `get_artifact` · `search_artifacts` · `share_artifact` · `add_feedback` · `summarize_feedback`

---

## Single tool

| Use case | Prompt example | Tool chain |
|---|---|---|
| Browse the gallery | "Show me all artifacts" | `list_artifacts` |
| Browse by type | "List all PDFs" / "Show HTML artifacts" | `list_artifacts(type=pdf\|html\|image)` |
| Browse by tag | "What has the 'design' tag?" | `list_artifacts(tag=design)` |
| Keyword search | "Find artifacts about dashboards" | `search_artifacts` |
| Publish from URL | "Publish this image: https://…" | `publish_artifact` |
| Publish with metadata | "Publish this HTML, call it 'Q4 Dashboard', tag it 'finance'" | `publish_artifact(title, tags, visibility)` |

---

## 2-step chains

| Use case | Prompt example | Tool chain |
|---|---|---|
| Look up artifact details | "Tell me about the Pikachu artifact" | `search_artifacts` → `get_artifact` |
| Look up by title directly | "Get me the landing page artifact" | `get_artifact(title=…)` → single result or candidate list |
| Look up by tag directly | "Find artifacts tagged 'finance' and show me details" | `get_artifact(tags=[…])` |
| Share a specific artifact | "Share the landing page artifact" | `share_artifact(query=…)` → link, or candidate list if ambiguous |
| Leave a review | "Give 4 stars to the data viz artifact" | `search_artifacts` → `add_feedback` |
| Read AI review summary | "What do people think about the chart artifact?" | `search_artifacts` → `summarize_feedback` |
| Publish then share immediately | "Publish this PDF and send me a link" | `publish_artifact` → `share_artifact` |
| Publish then annotate | "Publish this mockup, add a note saying it needs accessibility work" | `publish_artifact` → `add_feedback` |
| Check rating of latest artifact | "What's the feedback summary for the newest artifact?" | `list_artifacts(limit=1)` → `summarize_feedback` |

## Multi-step flows

| Use case | Prompt example | Tool chain |
|---|---|---|
| Publish → review → share | "Upload this mockup, rate it 5 stars, and share it with the team" | `publish_artifact` → `add_feedback` → `share_artifact` |
| Curated recommendations | "Find good HTML artifacts and pick the top-rated one" | `list_artifacts` → `get_artifact` → `summarize_feedback` |
| Peer review workflow | "Find the report artifact, review it, then show me the feedback summary" | `search_artifacts` → `get_artifact` → `add_feedback` → `summarize_feedback` |
| Share with expiry + notify | "Find the Q4 report, create a 48h link, give me the URL to paste in Slack" | `search_artifacts` → `share_artifact(expires_in_hours=48)` |
| Publish batch from URLs | "Publish these 3 images: [url1] [url2] [url3]" | `publish_artifact` ×3 (called once per URL in sequence) |
| Full end-to-end onboarding | "Publish my logo, add a description, then share it with my client" | `publish_artifact` → `add_feedback` → `share_artifact` |
| Browse → compare → pick best | "Show all PDFs, read the top 3, and tell me which has the best feedback" | `list_artifacts` → `get_artifact` ×3 → `summarize_feedback` ×3 |

## Human-in-the-loop (ambiguous inputs)

| Trigger | What Claude does | Tool behavior |
|---|---|---|
| Vague artifact name matches multiple | "Share the report" when 4 artifacts have 'report' in the title | `share_artifact` returns `candidates` list → Claude asks user to pick → re-calls with `artifact_id` |
| Title search returns multiple | "Get me the dashboard artifact" → 3 matches | `get_artifact` returns `candidates` list → Claude asks user to confirm |
| No artifact_id, no query | "Share an artifact" with nothing else | `share_artifact` returns clarification prompt → Claude asks for title or ID |

## Coverage gaps (no tool today)

| Missing use case | What would be needed | Workaround |
|---|---|---|
| Edit artifact metadata | `update_artifact` tool (PATCH `/artifacts/:id`) | Edit manually in the web UI |
| Delete an artifact | `delete_artifact` tool | Web UI only |
| List own private artifacts | `list_artifacts` doesn't thread user auth — public only | Web UI gallery |
| Upload a local file directly | MCP has no binary transport (protocol limit) | Host file publicly first → `publish_artifact(url=…)` |
| Upload image generated by Claude | Claude can't push bytes over MCP (protocol limit) | Save image to hosting URL → `publish_artifact(url=…)` |
| Delete / edit a comment | `delete_feedback` / `update_feedback` tools | No workaround today |
| Browse share links | `list_shares` tool | Call `get_artifact` → check `share_tokens` in response |
